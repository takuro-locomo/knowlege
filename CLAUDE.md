# 上野医院 ナレッジ管理システム

## このプロジェクトについて
上野医院（長野市・美容皮膚科）のための
美容医療ナレッジ収集・整理システム。
収集したナレッジはブログ・SNS・患者説明・スタッフ教育に活用する。

## フォルダ構成
- knowledge/01_脱毛/
- knowledge/02_シミ・色素沈着/
- knowledge/03_アンチエイジング・注射/
- knowledge/04_スキンケア・コスメ/
- knowledge/05_栄養・食事・サプリ/
- knowledge/06_ライフスタイル/
- knowledge/07_業界・トレンド/

## ナレッジノートのルール
1. ファイル名：YYYY-MM-DD_タイトル.md
2. 必ず以下の3セクションを含める
   - 【ファクト】医学的・科学的な事実
   - 【患者訴求】患者に刺さる言葉・場面
   - 【使える場面】ブログ/SNS/カウンセリング/スタッフ教育
3. 重複チェック：既存ノートと内容が似ていたらスキップ
4. 関連ノートがあればファイル名をリンクとして記載

## フォルダ振り分けルール
- 複数カテゴリにまたがる場合はメインフォルダに保存
  ＋関連フォルダに関連ノートとしてリンク記載
- 施術後ケア・日常習慣・栄養・コスメ成分も積極的に収集する
- 判断に迷ったら07_業界・トレンドへ

## カスタムコマンド

### /収集 [テーマ]
指定テーマのナレッジを収集してノートを作成する。

**使用例：** `/収集 脱毛` `/収集 ビタミンC 美肌` `/収集 アンチエイジング注射`

**実行手順（必ずこの順番で行うこと）：**

1. **Web検索（必須）** — 以下5クエリを必ず全て実行すること。検索をスキップしてはならない。
   - 日本語クエリ × 2（医療・科学情報寄り）
   - 英語クエリ × 2（海外最新研究寄り）
   - 「患者 悩み」視点クエリ × 1
   - 各クエリで上位3件を読み込む。医療ジャーナル・学術情報を優先する。

2. **ナレッジ分解** — 1記事から独立したナレッジを1〜3件抽出。各ナレッジは必ず以下を含む：
   - 【ファクト】医学的・科学的な事実
   - 【患者訴求】患者に刺さる言葉・悩み・場面
   - 【使える場面】ブログ/SNS/カウンセリング/スタッフ教育

3. **フォルダ判定** — フォルダ構成に従い最適なフォルダを選ぶ。複数カテゴリにまたがる場合はメインフォルダに保存。

4. **重複チェック** — 対象フォルダの既存ファイルと照合。類似ナレッジがあればスキップし関連リンクのみ追加。

5. **保存** — `knowledge/該当フォルダ/YYYY-MM-DD_タイトル.md` に保存。`weekly-summary/YYYY-MM-DD週.md` に1行追記。

**完了メッセージ形式：**
```
✅ 収集完了：X件のナレッジを追加しました
📁 振り分け先：〇〇フォルダ
🔗 関連ノート：△△（あれば）
⏭ 次回推奨テーマ：□□
```

### /ブログ [テーマ] [オプション]
指定テーマのブログ記事を生成し保存する。

**使用例：** `/ブログ 脱毛 夏向け` `/ブログ ピコシュア 40代女性`

**実行手順（必ずこの順番で行うこと）：**

1. **ナレッジ収集** — `knowledge/` 配下の全フォルダを検索し、テーマ・オプションに関連するナレッジノートを全て読み込む。

2. **素材統合** — 収集したノートから【ファクト】【患者訴求】【使える場面】を抜き出し、記事の素材として整理する。

3. **執筆** — 以下の条件を全て満たす記事を書く：
   - ブランドトーン：親しみやすく・専門的・押しつけがましくない
   - 構成：導入 → 悩み提示 → 解決策 → 施術説明 → アフターケア → まとめ
   - SEOキーワード（テーマ関連語）を見出しと本文に自然に含める
   - 文字数：1500〜2000字

4. **保存** — `knowledge/output/blog/YYYY-MM-DD_タイトル.md` に保存。

**完了メッセージ形式：**
```
✅ ブログ記事生成完了
📄 保存先：knowledge/output/blog/YYYY-MM-DD_タイトル.md
📝 文字数：〇〇字
🔑 使用SEOキーワード：〇〇、〇〇、〇〇
📚 参照ナレッジ：△△.md、△△.md
```

### /SNS [テーマ] [ターゲット]
指定テーマのSNS投稿文を3パターン同時生成し保存する。

**使用例：** `/SNS ピコシュア 40代女性` `/SNS 脱毛 夏前キャンペーン`

**実行手順（必ずこの順番で行うこと）：**

1. **ナレッジ収集** — `knowledge/` 配下から、テーマ・ターゲットに関連するナレッジノートを検索・読み込む。

2. **3パターン同時生成** — 以下を全て作成する：
   - **Instagram投稿文**：300字以内・改行多め・ハッシュタグ10〜15個付き
   - **X（Twitter）投稿文**：140字以内・簡潔でシェアされやすい表現
   - **Google口コミ返信テンプレート**：200字以内・感謝と施術の補足説明を含む

3. **トーン** — 患者の感情・悩みに寄り添う表現。過度な医療広告規制に引っかからない表現を使う（効果を断言しない）。

4. **保存** — `knowledge/output/sns/YYYY-MM-DD_タイトル.md` に保存。

**完了メッセージ形式：**
```
✅ SNS投稿文生成完了
📄 保存先：knowledge/output/sns/YYYY-MM-DD_タイトル.md
📚 参照ナレッジ：△△.md、△△.md
```

### /説明書 [施術名]
患者向け施術説明書を生成し保存する。

**使用例：** `/説明書 レーザー後アフターケア` `/説明書 脱毛初回カウンセリング`

**実行手順（必ずこの順番で行うこと）：**

1. **ナレッジ全収集** — `knowledge/` 配下から施術名に関連するナレッジノートを全て読み込む。

2. **執筆** — 以下の条件を全て満たす説明書を書く：
   - 読み手：患者（医療知識がない前提でわかりやすく）
   - 構成：施術概要 → 期待できる効果 → 当日の流れ → アフターケア → 注意事項 → よくある質問（Q&A形式）
   - 医学的に正確な表現を使う
   - 薬機法・医療広告ガイドラインに抵触しない表現を使う（効果の断定・比較優良表現を避ける）

3. **保存** — `knowledge/output/documents/YYYY-MM-DD_タイトル.md` に保存。

**完了メッセージ形式：**
```
✅ 患者向け説明書生成完了
📄 保存先：knowledge/output/documents/YYYY-MM-DD_タイトル.md
📚 参照ナレッジ：△△.md、△△.md
⚠️ 医師確認推奨箇所：〇〇（あれば）
```
